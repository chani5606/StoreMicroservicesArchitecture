# ADR-001: OrderService משתמש במסד נתונים רלציוני (SQL Server)

- סטטוס: מאושר
- תאריך: 2026-07-06

## הקשר

נתוני הזמנה כוללים ערכים כספיים (מחיר יחידה), מצבי מחזור חיים של הזמנה, ועדכונים שמגיעים מאירועים אסינכרוניים.

## החלטה

OrderService נשאר על SQL Server (רלציוני).

## נימוק

- נדרשות טרנזקציות ACID כדי לשמור על נכונות של מעברים כספיים ומצביים.
- עקביות חזקה עדיפה על eventual consistency בשמירת סטטוס הזמנה.
- מודל רלציוני מתאים לישויות טרנזקציוניות עם מגבלות ברורות.

## מסגרת CAP / עקביות

- בפועל זה מעדיף את המאפיינים של עקביות ו-partition tolerance שמצופים ממאגר רלציוני ראשי.
- זמינות בזמן partition נחלפת בנכונות של מצב ההזמנה שנשמר.

## השלכות

חיובי:
- הבטחות טרנזקציוניות חזקות למצב ההזמנה.
- שאילתות צפויות לתהליכי תפעול.

שלילי:
- פחות גמישות בסכימה מאשר אחסון דוקומנטי.
- סקיילינג אופקי מורכב יותר מאשר key-value/document.
