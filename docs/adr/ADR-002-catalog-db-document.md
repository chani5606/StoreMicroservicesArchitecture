# ADR-002: ProductCatalogService משתמש במסד דוקומנטים (MongoDB)

- סטטוס: מאושר
- תאריך: 2026-07-06

## הקשר

נתוני קטלוג יכולים להשתנות לפי קטגוריה ומתפתחים לעיתים קרובות (metadata של מוצר, שדות ייחודיים לקטגוריה).

## החלטה

ProductCatalogService משתמש ב-MongoDB כמסד דוקומנטים.

## נימוק

- מודל דוקומנט מתאים לתכונות מוצר משתנות בלי מיגרציות רלציוניות יקרות.
- מבנה דה-נורמליזציה ידידותי לקריאה מתאים לדפוסי שאילתות של קטלוג.
- התנהגות שמכוונת ל-BASE מקובלת בגלישה בקטלוג, כשסטייה קלה בעדכניות נסבלת.

## מסגרת CAP / עקביות

- המערכת מעדיפה זמינות ו-partition tolerance שמאפיינות מערכות דוקומנט מבוזרות.
- העקביות ניתנת לכיוונון ויכולה להיות eventual בעומסי קריאה גבוהים.

## השלכות

חיובי:
- סכימה גמישה ואיטרציה מהירה על metadata של מוצר.
- טיפול טבעי במסמכי JSON בתוך השירות.

שלילי:
- סמנטיקה טרנזקציונית חלשה יותר בהשוואה לזרמים רלציוניים ACID נוקשים.
